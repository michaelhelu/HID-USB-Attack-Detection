פרויקט: זיהוי מתקפת HID USB באמצעות Python וניתוח לוגים עם Sysmon
סקירה כללית

בפרויקט זה הדגמתי מתקפת HID (Human Interface Device) USB, שבה התקן USB זדוני מתחזה למקלדת ומבצע הקלדה אוטומטית של פקודות במערכת Windows.

מטרת הפרויקט הייתה להציג:

כיצד ניתן לבצע מתקפה באמצעות התקן HID.

כיצד ניתן לזהות פעילות חשודה במערכת באמצעות Sysmon.

כיצד ניתן לפתח מנגנון זיהוי בזמן אמת באמצעות Python.

כיצד ניתן לנתח את פעילות המערכת ולמפות אותה ל-MITRE ATT&CK.

הפרויקט בוצע כולו בסביבת מעבדה מבוקרת לצורכי לימוד בלבד.

סביבת העבודה

המערכת בה בוצע הניסוי:

מערכת הפעלה: Windows

כלי ניטור: Sysmon

שפת תכנות: Python

אוטומציה: PowerShell

התקן התקיפה:
DevKitty USB Nugget Microcontroller Platform Support for Adafruit CircuitPython

התקן זה מאפשר לבצע הזרקת הקשות מקלדת אוטומטית לאחר חיבורו למחשב.

שלב 1 – ביצוע מתקפת HID

בשלב הראשון יצרתי סקריפט עבור התקן ה-HID אשר מדמה מקלדת.

כאשר ההתקן מחובר למחשב, הוא מבצע באופן אוטומטי:

פתיחת חלון Run.

הרצת PowerShell.

הקלדה אוטומטית של פקודות במערכת.

הקוד ששימש לביצוע התקיפה נמצא בקובץ:

HIT_ATTACK.txt

סקריפט זה מדגים כיצד התקן USB זדוני יכול לבצע פעולות במערכת ללא אינטראקציה של המשתמש.

שלב 2 – הדגמת המתקפה

בפרויקט צירפתי מספר סרטונים המדגימים:

את ביצוע המתקפה בזמן אמת.

את התגובה של מערכת הזיהוי.

את אופן הפעולה של סקריפט הזיהוי.

הסרטונים מאפשרים לראות בצורה ברורה כיצד ההתקן מתחבר למערכת וכיצד מתבצעת ההזרקה האוטומטית של הפקודות.

שלב 3 – פיתוח מנגנון זיהוי באמצעות Python

לאחר ביצוע המתקפה, פיתחתי סקריפט Python שמנטר חיבור של התקני מקלדת חדשים למערכת.

הסקריפט משתמש ב-WMI על מנת לזהות אירוע של חיבור מקלדת חדשה.

כאשר מזוהה התקן HID מסוים, הסקריפט יכול:

להתריע על חיבור התקן חשוד

לבצע פעולה הגנתית (לדוגמה כיבוי המחשב)

הקוד נמצא בקובץ:

HID_Detector.py

מטרת הסקריפט היא להדגים כיצד ניתן ליישם מנגנון זיהוי בסיסי להתקני HID חשודים.

שלב 4 – הפעלת מנגנון הזיהוי באופן אוטומטי

כדי להבטיח שמנגנון הזיהוי יפעל בכל הפעלה של המחשב, הגדרתי Scheduled Task ב-Windows.

ה-Task מפעיל את סקריפט הזיהוי באופן אוטומטי בעת כניסת המשתמש למערכת.

כך ניתן להבטיח שהמערכת תמיד מנטרת חיבור של התקני HID.

שלב 5 – ניתוח לוגים באמצעות Sysmon

במהלך ביצוע המתקפה נאספו לוגים באמצעות Sysmon.

הלוגים נותחו במטרה לזהות:

יצירת תהליכים חשודים

הרצת PowerShell

פעולות שבוצעו על ידי הסקריפט

הניתוח המלא כולל:

צילומי מסך מתוך Event Viewer

הסבר לכל אירוע

מיפוי ל-MITRE ATT&CK

מסקנות מהחקירה

הניתוח נמצא בקובץ:

sysmon_analysis.docx
מיפוי למודל MITRE ATT&CK

במהלך הניתוח זוהו טכניקות הקשורות למודל MITRE ATT&CK.
